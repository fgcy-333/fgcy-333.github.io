Java数据类型教程 - Java char数据类型

char数据类型是16位无符号Java基元数据类型。它表示Unicode字符。

请注意，char是无符号数据类型。因此，char变量不能为负值。

字符数据类型的范围为0到65535，这与Unicode集的范围相同。

字符集和编码规则

我们经常会听到Unicode、UTF-8、UTF-16这些术语，然而，它们是完全不同的概念。Unicode是字符集，UTF-8、UTF-16是编码规则，具体概念如下：

  • 字符集：为每一个「字符」分配了一个唯一的 ID或者编号【这个编号称之为码点 Code Point】
  • 编码规则：将「编号」转换 为字节序列 的规则

外码和内码是计算机科学中常用的概念，举个例子来说明：

对于一个英文字母 “A”，在 ASCII 编码中它的外码是 65，在 Unicode 编码中它的外码是 U+0041。而对于计算机内部处理这个字符时，会将其转换成二进制形式的内码，比如在 ASCII 编码中 “A” 的内码是 01000001，在 Unicode 编码中 “A” 的内码可能是 00000000 01000001。

简单来说，外码是码点，而内码是计算机内部用于处理和存储字符的二进制编码方式。

ASCII字符集&编码

ASCII字符集：即美国信息交换标准码，分为标准ASCII码和扩展ASCII码。

  • 标准ASCII：标准ASCII字符集共有128个字符。【0-127】

  • 编码方式：标准ASCII编码是采用7位编码方式，以1个字节存储一个ASCII字符，字节最高位为0，即00000000-01111111或0x00-0x7F。

  • 扩展ASCII：由于标准ASCII字符集字符有限，往往无法满足实际需求，因此国际标准组织制定了在与标准ASCII规范相兼容的前提下将ASCII字符集扩充为8位代码的方法。每种扩充ASCII字符集可以扩充128个字符，这些扩充字符的编码均为 最高位为1 的8位代码。

    扩充的ASCII字符集即为扩展ASCII字符集，编码方式称为扩展ASCII编码。

  • 编码方式：常见的一种扩展ASCII为ISO-8859-1(也称为Latin-1)编码规范，用于支持部分欧洲语言。

GBXXXX字符集&编码

是指中国汉字编码字符集以及后续扩展的一系列规范，包括GB2312-80、GBK、GB18030。

GB2312-80

GB2312-80：中国国家标准简体中文字符集，包括了6763个常用汉字和682个全角的非汉字字符。它使用了两个字节表示一个汉字，其中第一个字节范围是a1-fe，第二个字节也是a1-fe，因此总共可以表示7445个常用简体中文字符，包括6763个汉字和682个非汉字符。

它只适用与中文字符，不包含繁体中文或者其他语言文字，因此在涉及多语言处理的场景下可能无法满足需求。随着时间的推移,gb2312已被gb18030所替代，在现代计算机系统中很少使用。

GBK

GBK：全称为《汉字内码扩展规范》。GBK兼容GB2312内码，是对GB2312内码的扩展，共收录了21003个汉字，883个字符，包含了简体、繁体中文以及日语、朝鲜语中的汉字。

编码方式：GBK编码方式采用双字节编码，占用两个字节的存储空间。gbk编码的设计目的是为了解决gb2312编码无法表示更多中文字符的问题，因此，gbk编码规则能够涵盖gb2312编码中所有的字符，并且加入了更多的中文字符。总的来说，gbk编码规则对于汉字的编码提供了较为全面的支持，但由于其只适用于中日韩语言，因此在全球范围内并不常见。现在常用的字符集包括utf-8、utf-16等，它们能够更好地适应不同语言的编码需求。

GB18030

GB18030：全称为《信息技术中文编码字符集》gb18030是一种中文字符编码标准，包含简体中文、繁体中文以及一些其他亚洲语言的字符。它的设计目的是为了能够同时兼容gbk、gb2312和unicode三个编码。

gb18030使用1-4个字节来表示不同的字符，其中大部分字符只需要2个字节，而少数较生僻的汉字则需要4个字节。它能够表示中国国内所有的汉字和中文符号，并且还支持日语、韩语等其他亚洲语言的字符集。与gb2312和gbk相比，gb18030有更大的容量，能够表示更多的字符。

此外，它还支持全角符号和半角符号混合，并且对于utf-8等unicode标准也有良好的兼容性。在网络通信、操作系统、办公软件等领域，gb18030被广泛应用。

编码方式：GB18030编码采用 变长 多字节 编码方式，包括1字节、2字节或4字节编码。

单字节编码与ASCII码相同，双字节编码与GBK编码相同并增加了一些扩展，四字节用于扩充汉字的编码

Unicode字符集&编码

Unicode字符集是为了统一所有语言的文字和符号而制定的编码字符集。

编码方式包括：UTF-8、UTF-16和UTF-32。

UTF-8编码：

UTF-8编码是针对Unicode字符集的 可变长度字符编码。UTF-8兼容ASCII字符编码，对于ASCII字符只占一个字节

对于其他字符占用多个字节，其第一个字节从最高位开始，打头连续的二进制位为1的个数决定了 编码占用的字节数，其余字节均以10开头，最多可占用6个字节。

UTF-8中可以用来 表示字符编码 的 实际位数 最多 为31位，即上表中x的个数；除去控制位剩余的x表示的位与Unicode中的码值是一一对应的，顺序也是一致的。

UTF-8带BOM与不带BOM：

UTF-8是与字节序无关的，UTF-8可以带BOM也可以不带BOM，UTF-8带BOM是在字节流前边增加3个字节（0xEFBBBF）的前缀，用来说明编码格式为UTF-8。UTF-8不带BOM才是标准形式，带BOM是微软的习惯，在其他操作系统中一般都是采用不带BOM的形式。

UTF-16编码：

针对Unicode字符集的编码方式，采用2字节或4字节长度编码UTF-16是一种Unicode字符集的编码方式，它使用16位来编码每个Unicode字符。Unicode字符集包含了世界上几乎所有的字符，包括不同语言的字母、数字、标点符号、符号等。

UTF-16编码可以表示Unicode字符集中从U+0000到U+10FFFF范围内的所有字符，其中U+0000到U+FFFF之间的字符以一个16位编码表示，而U+10000到U+10FFFF之间的字符需要用两个16位编码表示（也称为代理对）。

UTF-16编码有两种形式：大端序和小端序。在大端序中，高位字节位于低地址，而低位字节位于高地址；在小端序中，情况相反。因此，当计算机存储UTF-16编码时，需要指定是采用大端序还是小端序。

在实际应用中，由于UTF-8编码具有更好的兼容性和节省空间的优势，UTF-8已经成为事实上的标准，但UTF-16仍然被广泛使用，特别是在Windows操作系统和Java程序中。

ANSI编码

 ANSI编码是指系统默认的编码方式，Windows可以在命令窗口中通过chcp指令查看系统默认编码。

  • 简体中文系统：ANSI编码为GBK或GB2312。
  • 英文系统：ANSI编码为ASCII码。
  • 繁体中文系统：ANSI编码为BIG-5。
